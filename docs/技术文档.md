# LLM智能记账系统 - 技术文档

## 1. 系统架构设计

### 1.1 整体架构
本系统采用前后端分离架构，前端使用Vue 3构建单页面应用，后端使用Spring Boot构建RESTful API服务。

```
┌──────────────┐     HTTPS      ┌──────────────────┐
│   Vue 3 前端  │ ←───────────→ │  Spring Boot 后端 │
│  (端口 3000) │   JSON/REST    │   (端口 8000)    │
└──────────────┘                └────────┬─────────┘
       ↓                                    ↓
┌──────────────┐                    ┌──────────────┐
│   浏览器缓存   │                    │    MySQL     │
│   (localStorage)│                │  (数据库)     │
└──────────────┘                    └──────────────┘
                                            ↓
                                    ┌──────────────┐
                                    │    Ollama    │
                                    │ (本地大模型)  │
                                    │ (端口 11434) │
                                    └──────────────┘
```

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 | 渐进式JavaScript框架 |
| 前端UI | Naive UI | Vue 3组件库 |
| 图表库 | ECharts | 数据可视化 |
| 后端框架 | Spring Boot 3.3.5 | 快速开发框架 |
| ORM | Spring Data JPA | 数据持久化 |
| 安全 | Spring Security + JWT | 身份认证 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 大模型 | Ollama + Qwen3:8B | 本地LLM服务 |

## 4. 核心功能设计

### 4.1 LLM智能解析流程

```
用户输入 → OCR图片识别 → 自然语言解析 → 结构化数据提取 → 重复检测 → 入库

详细流程:
1. 用户输入自然语言描述或上传票据图片
2. 如果是图片，使用Tesseract.js进行OCR识别
3. 将文本发送给Ollama大模型
4. 模型解析出结构化的账单信息(金额、描述、分类、类型)
5. 系统进行重复账单检测
6. 如果有重复，提醒用户确认
7. 确认后保存到数据库
```

### 4.2 家庭账本权限控制

```
角色层级: OWNER > ADMIN > MEMBER > VIEWER

权限矩阵:
| 功能           | OWNER | ADMIN | MEMBER | VIEWER |
|----------------|-------|-------|--------|--------|
| 查看交易       | ✓     | ✓     | ✓      | ✓      |
| 创建交易       | ✓     | ✓     | ✓      | ✗      |
| 修改自己的交易 | ✓     | ✓     | ✓      | ✗      |
| 修改他人的交易 | ✓     | ✓     | ✗      | ✗      |
| 删除自己的交易 | ✓     | ✓     | ✓      | ✗      |
| 删除他人的交易 | ✓     | ✓     | ✗      | ✗      |
| 管理成员       | ✓     | ✓     | ✗      | ✗      |
| 删除家庭账本   | ✓     | ✗     | ✗      | ✗      |
```

### 4.3 重复账单检测算法

```
检测条件:
1. 时间范围: 最近30天
2. 金额误差: < 0.01元
3. 描述相似度: ≥ 60% (使用编辑距离算法)

算法步骤:
1. 获取用户最近30天的所有交易
2. 对于每个新交易:
   a. 计算金额相似度
   b. 计算描述相似度(编辑距离)
   c. 如果同时满足金额和描述相似，返回重复警告
3. 展示给用户确认
```

## 5. 统计与可视化

### 5.1 统计指标

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 总支出 | Σ(所有expense类型金额) | 累计支出 |
| 总收入 | Σ(所有income类型金额) | 累计收入 |
| 结余 | 总收入 - 总支出 | 财务健康度 |
| 储蓄率 | (结余 / 总收入) × 100% | 储蓄能力 |
| 健康分 | 基于收入支出比计算 | 0-100分 |

### 5.2 可视化图表

1. **资金流向桑基图**: 展示收入来源和支出流向
2. **分类饼图**: 展示各分类占比
3. **月度趋势图**: 展示收支变化趋势
4. **周期性账单**: 展示订阅支出

## 6. 安全设计

### 6.1 JWT认证流程

```
1. 用户登录 → 后端验证密码 → 生成JWT Token
2. 前端保存Token到localStorage
3. 后续请求在Header中携带: Authorization: Bearer <token>
4. 后端Filter验证Token有效性
5. 验证通过后，将用户信息注入SecurityContext
```

### 6.2 密码安全

- 使用BCryptPasswordEncoder加密
- 密码强度要求: 至少6位
- Token有效期: 24小时

## 7. 部署说明

### 7.1 环境要求

- JDK 17+
- Node.js 18+
- MySQL 8.0+
- Ollama 0.1+

### 7.2 启动步骤

1. **启动MySQL数据库**
2. **启动Ollama服务**
   ```bash
   ollama serve
   ```
3. **下载大模型**
   ```bash
   ollama pull qwen3:8b
   ```
4. **启动后端**
   ```bash
   cd backend-springboot
   mvn spring-boot:run
   # 或
   java -jar target/backend-springboot-1.0.0.jar
   ```
5. **启动前端**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
